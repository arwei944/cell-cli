# 智能体自举式开发工作台 - 开发任务清单

> 版本: v1.0
> 状态: 待确认
> 目标: 构建完全自主、人类零参与的智能体开发流水线

---

## 总览

| 层级 | 模块数 | 任务数 | 优先级 |
|------|--------|--------|--------|
| P0 核心自主能力 | 4 | 24 | 必须先做 |
| P1 增强能力 | 4 | 18 | 核心完成后做 |
| P2 优化能力 | 2 | 8 | 最后做 |
| **总计** | **10** | **50** | - |

## P0 - 核心自主能力（必须先做）

### 模块 1：Agent Onboarding 服务（一键就绪）

> 目标：从 `git clone` 到"可以开始开发"，全自动，零配置，幂等。

| 任务 ID | 任务名称 | 任务描述 | 前置依赖 | 验收标准 | 涉及文件 |
|---------|---------|---------|---------|---------|---------|
| ONB-001 | 环境检测服务 | 自动检测 Rust、cargo、git、node 等开发依赖是否安装，输出检测报告 | 无 | 1. `cell dev doctor` 命令可用<br>2. 能正确检测已安装/未安装的工具<br>3. 输出清晰的检测报告（通过/警告/失败）<br>4. 缺失的工具给出安装建议 | `src/application/onboarding_service.rs`<br>`src/interfaces/commands/dev_cmd.rs` |
| ONB-002 | 依赖自动安装 | 检测到缺失的依赖时，自动安装（rustup、cargo 包等） | ONB-001 | 1. 缺失 Rust 时自动提示安装命令<br>2. 缺失 cargo 包时自动安装<br>3. 安装失败给出明确错误信息<br>4. 安装成功后自动验证 | 同 ONB-001 |
| ONB-003 | 工具链自动构建 | 自动 `cargo build --release` 构建 cell CLI | ONB-001 | 1. 检测到未构建时自动构建<br>2. 构建失败输出错误日志<br>3. 构建成功后验证二进制可用<br>4. 增量构建（已构建就跳过） | 同 ONB-001 |
| ONB-004 | Git Hooks 自动安装 | 自动安装 pre-commit / pre-push 钩子 | ONB-003 | 1. 自动检测钩子是否已安装<br>2. 未安装时自动调用 `cell enforcement install-hooks`<br>3. 安装成功后验证钩子存在 | 同 ONB-001 |
| ONB-005 | Agent 自动注册 | 自动为当前智能体注册身份，分配角色 | ONB-003 | 1. 自动生成 Agent ID 和名称<br>2. 自动注册为 Developer 角色<br>3. 已注册的 Agent 不重复注册<br>4. 注册信息持久化存储 | 同 ONB-001 + `agent_protocol_service.rs` |
| ONB-006 | 基线自动建立 | 自动建立熵值基线、架构快照 | ONB-003 | 1. 首次运行自动计算熵值并存为基线<br>2. 已有基线则跳过<br>3. 自动生成架构快照 | 同 ONB-001 + `entropy_service.rs` |
| ONB-007 | 就绪报告生成 | 输出完整的就绪报告，包括环境状态、Agent 身份、架构状态、下一步建议 | ONB-001 ~ ONB-006 | 1. 报告包含：环境检测结果、Agent 信息、架构状态、熵值、待处理任务<br>2. 给出明确的"下一步建议"<br>3. 格式清晰，一眼能看懂 | 同 ONB-001 |
| ONB-008 | `cell dev start` 命令 | 一键启动命令，串联所有 onboarding 步骤 | ONB-007 | 1. 一条命令完成所有准备工作<br>2. 幂等：多次运行结果一致<br>3. 容错：某步失败不影响其他步骤<br>4. 进度显示：每一步在干什么清晰可见 | `src/interfaces/commands/dev_cmd.rs` |

> 目标：Agent 自己知道该干什么，不用人分配任务。

| TSK-003 | Issue 同步（可选） | 从 GitHub/Gitee 同步 Issues 为任务 | TSK-001 | 1. 支持从 GitHub Issue 导入<br>2. 支持标签映射（bug/enhancement 等）<br>3. 增量同步，不重复创建 | 同 TSK-001 |
| TSK-004 | 任务优先级排序 | 自动按优先级、依赖关系、紧急度排序 | TSK-001 | 1. P0 > P1 > P2 基础排序<br>2. 考虑依赖关系（被依赖的先做）<br>3. 考虑预估工作量（小任务优先）<br>4. 可配置排序策略 | 同 TSK-001 |

| TSK-006 | 任务池管理 | 统一管理所有来源的任务 | TSK-001 ~ TSK-005 | 1. `cell task list` 命令可用<br>2. 支持按优先级/状态/来源筛选<br>3. 支持认领任务、完成任务<br>4. 任务状态持久化 | 同 TSK-001 + `agent_protocol_service.rs` |
| TSK-007 | 下一个任务推荐 | 自动推荐"现在最该做的任务" | TSK-006 | 1. `cell task next` 命令可用<br>2. 推荐 1 个最高优先级、无依赖、未认领的任务<br>3. 给出推荐理由<br>4. 可一键认领 | 同 TSK-001 |

### 模块 4：自我验证与修复循环

> 目标：Agent 写的代码，自己验证、自己修，不需要人来管。

| 任务 ID | 任务名称 | 任务描述 | 前置依赖 | 验收标准 | 涉及文件 |
|---------|---------|---------|---------|---------|---------|
| VRF-001 | 架构自检与自动修复 | 写完代码自动跑 lint，违规了自动修 | 现有 `arch_fixer` | 1. `cell verify self` 命令可用<br>2. 自动调用 `cell arch lint --fix`<br>3. 修复后重新验证<br>4. 修不好的标记出来 | `src/application/self_verify_service.rs` |
| VRF-002 | 测试自检 | 自动运行测试，失败了收集错误信息 | 无 | 1. 自动运行 `cargo test`<br>2. 失败时收集错误日志<br>3. 按失败原因分类（编译错误/断言失败/超时）<br>4. 输出结构化的失败报告 | 同 VRF-001 |
| VRF-003 | 熵值退化检测与预警 | 自动检测熵值是否退化 | 现有 `entropy_service` | 1. 对比基线熵值<br>2. 退化超过阈值触发预警<br>3. 给出退化最严重的模块<br>4. 给出优化建议方向 | 同 VRF-001 |
| VRF-004 | 编译错误自动修复（基础版） | 简单的编译错误自动修复 | VRF-002 | 1. 能修复：未使用的 import、拼写错误的变量、类型不匹配（简单情况）<br>2. 修复后自动重新编译验证<br>3. 最多重试 3 次<br>4. 修不好的标记为"需要人工介入" | 同 VRF-001 |
| VRF-005 | 验证-修复循环引擎 | 自动跑"验证→修复→验证"循环，直到通过或达到最大重试次数 | VRF-001 ~ VRF-004 | 1. `cell agent verify` 命令可用<br>2. 自动串联：架构检查 → 测试 → 熵值检查<br>3. 失败了自动尝试修复<br>4. 最多重试 N 次（可配置，默认 3）<br>5. 输出每一步的结果 | 同 VRF-001 |
| VRF-006 | 自动回滚机制 | 修复失败时自动回滚到稳定状态 | VRF-005 | 1. 检测到"越修越坏"时自动回滚<br>2. 回滚到上一个通过验证的 commit<br>3. 回滚后生成问题报告<br>4. 标记任务为"需要人工介入" | 同 VRF-001 + `git_integration_service.rs` |

## 验收总标准

### 整体验收标准

1. **完全无人干预**：从 `cell dev start` 开始，到任务完成，不需要人类输入任何指令
2. **可追溯**：每一步操作都有日志，出了问题能定位到原因
3. **幂等安全**：同一条命令跑多少次结果都一样，不会搞坏环境
4. **故障自愈**：遇到问题优先自己解决，实在不行才标记为"需要人工介入"
5. **自举验证**：工具链自己的代码，必须通过自己的所有检查

### 质量门禁（必须全部通过）

- [ ] 架构零违规（`cell arch lint` 全通过）
- [ ] 熵值不退化（对比基线不上升）
- [ ] 测试全通过（`cargo test` 0 失败）
- [ ] 代码审查通过（`cell review --deep` 无严重问题）
- [ ] Git Hooks 生效（pre-commit / pre-push 能正确阻断违规提交）
