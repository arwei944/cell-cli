> 生成时间: 2026-06-26

## 1. 架构概览

Cell Architecture 采用经典的四层架构，严格遵循依赖倒置原则。

```
┌─────────────────────────────────────────────────────────────┐
│                    Interfaces 层（接口层）                    │
│  CLI / WebSocket / HTTP API                                 │
│  11 命令组 / 45+ 子命令                                      │
└─────────────────────────────────────────────────────────────┘
                            │ 调用
┌─────────────────────────────────────────────────────────────┐
│                    Adapters 层（适配器层）                    │
│  文件适配器 / Git 适配器 / 模板引擎 / 代码生成器              │
│  实现端口接口，隔离外部依赖                                   │
└─────────────────────────────────────────────────────────────┘
                            │ 通过端口调用
┌─────────────────────────────────────────────────────────────┐
│                  Application 层（应用层）                     │
│  40+ 服务：编排业务流程，协调领域对象                          │
│  无业务逻辑，纯用例编排                                       │
└─────────────────────────────────────────────────────────────┘
                            │ 调用
┌─────────────────────────────────────────────────────────────┐
│                    Domain 层（领域层）                        │
│  30+ 模块：核心业务逻辑、领域模型、规则引擎                    │
│  零外部依赖，最稳定层                                         │
└─────────────────────────────────────────────────────────────┘
```

## 3. 架构规则验证

### 3.1 依赖方向

```
Interfaces → Adapters → Application → Domain
```

**验证结果:**
- ✅ 无循环依赖
- ✅ 领域层零外部依赖
- ✅ 应用层仅依赖领域层
- ✅ 适配器层通过端口接口解耦

### 3.2 模块职责

| 层 | 职责 | 禁止事项 |
|----|------|----------|
| Domain | 核心业务逻辑、领域模型 | 禁止依赖外部库 |
| Application | 用例编排、服务协调 | 禁止业务逻辑泄露 |
| Adapters | 外部交互、协议实现 | 禁止领域逻辑 |
| Interfaces | CLI/API 入口 | 禁止直接访问领域层 |

### 3.3 架构违规

- **当前违规数**: 0
- **警告数**: 0
- **严重程度**: 无
